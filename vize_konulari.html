1
Temel NYP Kavramları
📦
Nesne & Sınıf
Sınıf: veri + davranış şablonu. Nesne: sınıfın bellekteki somut örneği. BankaHesabi h1; → nesne başlatma.
🔒
Kapsülleme
Veri gizleme. private alanlara doğrudan erişilemez; public metodlar üzerinden erişilir.
🏠
Sınıf İçi Erişim
Üye metodlar private alanlara doğrudan ulaşır. this->alan ile nesne kendine başvurur.
🚪
Sınıf Dışı Erişim
public metod zorunlu. h1.getGelir() doğru; h1.gelir (private ise) hata verir.
Soru Tipi: Boşluk Doldurma / Kısa Cevap
private ve public arasındaki fark nedir?
Bir nesneyi nasıl başlatırsınız?
this işaretçisi ne anlama gelir?
2
Yapıcı ve Yıkıcı Fonksiyonlar
🔨
Default Constructor
Parametresiz. BankaHesabi h; ile nesne oluşturulursa çağrılır. Alanları sıfırlar.
⚙️
Parametreli Constructor
Farklı imzalar → aşırı yükleme. Top(int r), Top(int r, string renk) gibi.
📋
Copy Constructor
Hesap(const Hesap& diger). Nesne kopyalanırken çağrılır. Derin / sığ kopya farkı önemli.
🗑️
Destructor
~BankaHesabi(). Nesne kapsam dışına çıkınca otomatik çağrılır. Bellek temizleme.
class BankaHesabi { private: string hesapNo; double gelir, gider; public: BankaHesabi(); // default constructor BankaHesabi(string no); // parametreli BankaHesabi(string no, double g); // aşırı yüklenmiş ~BankaHesabi(); // yıkıcı };
Soru Tipi: Yapıcı Fonksiyon Tasarımı
Vadeli / vadesiz / yatırım hesabı — her biri farklı alanlarla başlatılır → farklı yapıcı imzaları tasarlayın
Top sınıfı örneğindeki gibi: Top(), Top(int), Top(int, string) yazın
3
Fonksiyon Aşırı Yüklemesi
🏦
Banka Hesabı Örneği
Farklı hesap türleri farklı initialize() imzası. Vadeli: faiz oranı alır. Vadesiz: almaz. Yatırım: enstrüman türü alır.
🎮
Oyun Motoru Örneği
spawn(int x, int y), spawn(Vektor2D pos), spawn(string sahne) — aynı isim, farklı parametreler.
💡 Aşırı yükleme kararını derleyici verir: parametreler farklıysa farklı fonksiyon çağrılır. Dönüş tipi tek başına yeterli değildir.
4
Operatör Aşırı Yüklemesi
⚡ Unary Operatörler
!
Mantıksal DEĞİL — hesap geçerli mi? !h1 → bakiye sıfırsa true
−
Tekli eksi — koordinatın negatifi. -v → (-x, -y). Çıkarma değil!
++
Artırma — prefix ++h ve postfix h++ farklı imza! Postfix int parametresi alır.
−−
Azaltma — yukarıdaki ile aynı kural geçerli.
🔗 Binary Operatörler
+
temp.gelir = h1.gelir + h2.gelir — varlık birleştirme
−
temp.gider = h1.gider - h2.gider
∗
Çarpma — örn. faiz hesabı: h1 * faizOrani
/
Bölme — ortalama veya pay/payda senaryosu
< >
Bakiye karşılaştırma: h1 < h2
==
Eşitlik: aynı hesap numarası ve bakiye kontrolü
// Banka Hesabı — operator+ (VARLIKLARIM senaryosu) BankaHesabi operator+(const BankaHesabi& diger) const { BankaHesabi temp; temp.gelir = this->gelir + diger.gelir; temp.gider = this->gider + diger.gider; return temp; } // Tekli eksi — koordinat örneği Koordinat operator-() const { return Koordinat(-x, -y); } // Prefix ++ BankaHesabi& operator++() { ++gelir; return *this; } // Postfix ++ (int parametresi prefix'ten ayırt eder) BankaHesabi operator++(int) { BankaHesabi eski = *this; ++gelir; return eski; }
Soru Tipi: Operatör Tanımlama
VARLIKLARIM: h1 + h2 + h3 → tüm hesapların gelir/gider toplamı
Prefix ++ ile postfix ++ imza farkını gösterin
Tekli − operatörünü koordinat sınıfında yazın
